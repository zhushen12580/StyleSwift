## 一、对比度原则

### 文字可读性

```
WCAG 标准对比度要求：
- 普通文本：至少 4.5:1
- 大号文本（18px+ 或 14px+ 加粗）：至少 3:1
- AAA 级别：普通文本 7:1，大号文本 4.5:1
```

### 对比度计算

```javascript
// 相对亮度计算
function getLuminance(r, g, b) {
  const [rs, gs, bs] = [r, g, b].map(c => {
    c = c / 255;
    return c <= 0.03928 ? c / 12.92 : Math.pow((c + 0.055) / 1.055, 2.4);
  });
  return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs;
}

// 对比度计算
function getContrastRatio(l1, l2) {
  const lighter = Math.max(l1, l2);
  const darker = Math.min(l1, l2);
  return (lighter + 0.05) / (darker + 0.05);
}
```

### 常见背景色推荐前景色

| 背景色 | 推荐前景色 | 对比度 |
|--------|-----------|--------|
| #FFFFFF | #333333 | 12.6:1 |
| #FFFFFF | #666666 | 5.7:1 |
| #1A1A1A | #E0E0E0 | 14.4:1 |
| #1A1A1A | #B0B0B0 | 7.3:1 |
| #007BFF | #FFFFFF | 4.5:1 |

## 二、视觉层级

### 字体大小层级

```
推荐比例：1.25 - 1.5 (Major Third)

基础字号：16px
- H1: 32px - 40px (2x - 2.5x)
- H2: 24px - 28px (1.5x - 1.75x)
- H3: 20px - 22px (1.25x - 1.375x)
- Body: 16px (1x)
- Small: 14px (0.875x)
- Caption: 12px (0.75x)
```

### 间距层级

```
8px 基准网格系统：
- xs: 4px  (0.5x)
- sm: 8px  (1x)
- md: 16px (2x)
- lg: 24px (3x)
- xl: 32px (4x)
- 2xl: 48px (6x)
```

### 视觉权重

```
影响视觉权重的因素：
1. 大小：越大越重要
2. 颜色：越亮/饱和度越高越引人注意
3. 对比：与周围元素的差异度
4. 位置：F型阅读模式，左上角最重要
5. 留白：周围留白越多越突出
```

## 三、留白原则

### 呼吸感

```
内容区域间距建议：
- 段落间距：1.5x 行高
- 区块间距：24px - 48px
- 页面边距：16px - 32px（移动端），48px - 96px（桌面端）
```

### 密度控制

```
避免信息过载：
- 每行字符数：45-75 字符（最佳可读性）
- 段落行高：1.5 - 1.8
- 表单字段间距：至少 16px
```

## 四、一致性原则

### 颜色一致性

```
一套配色方案应包含：
- 主色 (Primary): 1 种，用于品牌识别、主要操作
- 辅助色 (Secondary): 1-2 种，用于次要信息
- 强调色 (Accent): 1 种，用于重要提示、CTA
- 中性色 (Neutral): 3-5 种，用于文字、边框、背景
- 状态色 (Status): 成功/警告/错误/信息
```

### 圆角一致性

```
圆角系统：
- 无圆角：0px（硬朗、现代）
- 小圆角：4px（微妙柔和）
- 中圆角：8px（平衡）
- 大圆角：12px - 16px（友好、柔和）
- 圆形：50%（按钮、头像）
```

### 阴影层级

```css
/* 阴影系统 */
--shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
--shadow-md: 0 4px 6px rgba(0, 0, 0, 0.1);
--shadow-lg: 0 10px 15px rgba(0, 0, 0, 0.1);
--shadow-xl: 0 20px 25px rgba(0, 0, 0, 0.15);
```

## 五、深色模式设计原则

### 不要简单反转颜色

```
错误做法：
- 直接将白色背景转为黑色
- 直接将黑色文字转为白色

正确做法：
- 深色背景：#1A1A1A ~ #2D2D2D（不是纯黑）
- 浅色文字：#E0E0E0 ~ #F5F5F5（不是纯白）
- 降低对比度，避免刺眼
```

### 深色模式配色建议

```css
/* 深色模式调色板 */
--bg-primary: #1A1A1A;     /* 主背景 */
--bg-secondary: #2D2D2D;   /* 卡片、侧边栏背景 */
--bg-tertiary: #3D3D3D;    /* 悬停状态、输入框背景 */
--text-primary: #E0E0E0;   /* 主要文字 */
--text-secondary: #A0A0A0; /* 次要文字 */
--text-muted: #707070;     /* 禁用、提示文字 */
--border: #404040;         /* 边框 */
--divider: #333333;        /* 分割线 */
```

### 深色模式注意事项

```
1. 降低饱和度：高饱和度在深色背景下更刺眼
2. 保持层次感：通过不同的灰度区分层级
3. 图片处理：考虑添加轻微透明度或边框
4. 阴影调整：深色模式下阴影效果减弱，考虑用发光代替
```

## 六、响应式设计原则

### 断点系统

```
常见断点：
- Mobile: < 576px
- Tablet: 576px - 768px
- Desktop: 768px - 1024px
- Large: 1024px - 1440px
- XLarge: > 1440px
```

### 触控友好

```
触控目标最小尺寸：
- 最小点击区域：44px × 44px (Apple)
- 最小点击区域：48px × 48px (Material Design)
- 按钮间距：至少 8px
```
