网页样式美化器 Chrome 插件需求文档

## 1. 项目背景

随着互联网的发展,用户对网页美观度的要求越来越高。然而,许多网站的设计仍然简陋,无法满足现代用户的审美需求。为解决这一问题,我们提出开发一个Chrome插件,允许用户一键美化任何网页的样式。

## 2. 项目目标

开发一个Chrome插件,能够让用户快速、简便地改变任何网页的视觉样式,以满足个人审美需求，并集成 AI 技术智能生成和优化样式。

## 3. 功能需求

### 3.1 核心功能

1.  **此站点模式 (Site Mode)** - *核心实现*
    *   提供多种预设风格模板（例如：`modern`, `retro`, `eyecare`, `cute`）。用户选择后，结合页面结构信息，由 AI 生成完整 CSS。
    *   AI 生成自定义样式：用户在 `popup.html` 提供的文本框中输入对期望风格的描述，该描述将与页面结构一同发送给后端 AI，生成相应的 CSS 代码。
    *   允许直接输入 CSS 代码进行深度定制：用户可以在 `popup.html` 提供的文本区域直接编写或粘贴 CSS 代码。
    *   样式应用后，会生成唯一的 `style_id`，并将样式代码及 `style_id` 通过 `chrome.storage.local` 存储，以当前网站的 hostname 为键，实现持久化和自动应用。
    *   实时预览样式应用效果：样式直接在当前激活的标签页上应用。

2.  **细节模式 (Details Mode)** - *核心实现*
    *   提供元素选择工具 (`content.js` 实现)：用户点击 "定位元素" 按钮后，可在目标网页上通过鼠标悬停高亮、点击选择特定 HTML 元素。
    *   捕获元素信息：选定元素后，`content.js` 会提取其 CSS 路径 (selector)、现有计算样式、HTML 结构等，并通过 `chrome.storage.local` (`selectedElement`) 暂存。
    *   AI 辅助调整单个元素样式：用户针对选定元素输入样式描述 (如"将此按钮变为红色")，结合捕获的元素信息，发送至后端 AI (`/api/generate_element_style`) 生成该元素的 CSS。
    *   样式合并与应用：
        *   后端会尝试将新生成的元素样式与当前页面已应用的整体样式（如果有）进行合并，处理潜在的规则冲突或重复。
        *   合并后的完整 CSS (包含对特定元素的修改) 会被发送回前端，并应用到页面。
        *   更新后的完整样式及 `style_id` 会保存在 `chrome.storage.local` 中，键为当前网站 hostname。
    *   **注意**: "保存元素级样式配置供重复使用" 目前的实现是将元素修改融入整个页面的样式配置中，而非独立的、可复用的元素级片段。

3.  **挂件模式 (Widget Mode)** - *初步规划/UI占位，核心逻辑待开发*
    *   `popup.html` 中已包含 "全站点模式 (All Sites)" 的 UI 入口及挂件选择器 (`widgetSelector`)、自定义代码输入区 (HTML/CSS/JS 标签页)。
    *   **当前状态**:
        *   核心的 AI 生成定制挂件逻辑、代码注入和跨网站同步功能尚未在 `background.js` 或 `app.py` 中实现。
        *   主要作为未来功能的规划。
    *   **规划功能**:
        *   提供常用功能挂件库（天气、便签、计时器等）。
        *   AI生成定制挂件：用户描述需求→生成HTML/CSS/JS代码。
        *   支持直接输入代码创建自定义挂件。
        *   挂件可固定显示在浏览器侧边栏或浮动于页面指定位置。
        *   跨网站挂件同步显示（需用户授权）。

4.  **AI 样式生成 (基础能力)** - *核心实现*
    *   提取网页 DOM 结构 (`content.js` 中的 `getPageStructure` 函数) 和关键 CSS 信息，发送至后端 AI 模型。
    *   后端 (`app.py`) 根据用户偏好（选择的风格或自定义描述）和页面信息，构建提示词 (prompt)。
    *   集成第三方 AI 大模型：
        *   主要使用 **Deepseek AI API** (通过 `openai` 库配置)。
        *   备用 **Claude AI API** (通过 `api.link-ai.tech` 代理访问)。
        *   包含 API 调用重试逻辑。
    *   AI 返回的 CSS 会经过初步处理 (`extract_css_from_response` 函数) 来提取有效的 CSS 代码。
    *   生成的样式会与 `style_id`、URL 等信息一同存储在后端 MySQL 数据库的 `styles` 表中。
    *   **安全性**: "自动验证生成代码的安全性" 目前主要依赖于 CSS 提取逻辑，可考虑引入更专业的 CSS 清理库。

### 3.2 扩展功能

1.  **自定义样式微调与创建**
    *   用户可以通过 "直接输入 CSS 代码" 功能完全控制样式。
    *   AI 生成的样式结果也可以被用户复制，在外部编辑后再通过 "直接输入 CSS" 应用。
    *   （未来可考虑）提供一个更友好的 UI 在插件内直接编辑和微调 AI 生成的 CSS。

2.  **样式分享** - *未来规划*
    *   用户可以分享自己创建的样式。
    *   其他用户可以下载和使用分享的样式。

3.  **网页类型识别** - *未来规划*
    *   自动识别当前网页类型(如新闻、社交媒体、电商等)。
    *   根据网页类型推荐适合的样式。

4.  **性能优化 (样式缓存)** - *已实现*
    *   通过 `chrome.storage.local` 缓存已应用的样式。当用户重复访问已应用过样式的网站时，`content.js` 会自动从本地存储加载并应用样式，无需重新生成。

## 4. 技术需求

1.  **Chrome插件开发 (Manifest V3)**
    *   使用 HTML, CSS, JavaScript (ES6+) 开发插件界面 (`popup.html`, `popup.js`) 和后台服务 (`background.js` - Service Worker) 及内容脚本 (`content.js`)。
    *   遵循 Manifest V3 规范和最佳实践。

2.  **网页结构分析**
    *   `content.js` 中的 `getPageStructure` 模块负责提取页面主要 DOM 结构、视口信息及部分关键 CSS。

3.  **AI 集成**
    *   后端 Flask (`app.py`) 应用负责与 AI 大模型交互。
    *   使用 `OpenAI` Python 库调用 Deepseek API。
    *   使用 `requests` 库调用 Claude API (通过 `api.link-ai.tech` 代理)。
    *   提示词工程 (Prompt Engineering) 在 `app.py` 中实现，根据用户输入和页面上下文动态构建提示。

4.  **样式渲染**
    *   `content.js` 负责动态创建 `<style>` 标签 (ID: `beautifier-style`) 并将 CSS 注入到当前网页，实现样式覆盖。
    *   确保新样式不会破坏网页的原有功能（依赖于 AI 生成 CSS 的质量和合理的 CSS 选择器）。

5.  **安全性**
    *   **前端**: `manifest.json` 中应配置合理的内容安全策略 (CSP)。
    *   **后端**:
        *   AI 生成的 CSS 内容在存入数据库或返回给前端前，会通过 `extract_css_from_response` 进行基础清理。
        *   （待增强）考虑引入更健壮的 HTML/CSS 清理库以防止潜在的 XSS 或样式注入问题。
    *   **注意**: AI 模型的调用发生在后端服务器，并非完全在用户本地。

## 5. 用户界面需求

### 1. 插件弹出窗口 (`popup.html`)
    *   简洁明了的界面设计，使用 Tailwind CSS 辅助样式。
    *   模式选择：支持 "此站点模式"、"细节模式"、"全站点模式" (挂件模式入口)。
    *   样式选择：
        *   预设样式下拉列表。
        *   自定义 AI 描述输入框。
        *   自定义 CSS 输入文本域。
    *   细节模式下：包含 "定位元素" 按钮及元素样式描述输入框。
    *   操作按钮："一键美化" / "应用样式" 等。
    *   加载指示器：在 AI 处理等耗时操作时显示。

### 2. 设置页面与管理 (主要集成于弹出窗口)
    *   用户偏好设置：主要通过弹出窗口中的样式及模式选择实现。
    *   自定义样式编辑器：即弹出窗口中提供的自定义 CSS 输入框和 AI 描述输入框。
    *   样式管理：
        *   当前实现：应用新样式会覆盖之前为该站点保存的样式。`style_id` 用于标识。
        *   （未来规划）可以考虑更显式的样式保存、命名、列表管理、删除、分享等功能。

## 6. 性能需求

1.  样式转换时间：目标控制在用户可接受范围内（例如，简单页面和快速AI响应下几秒内），复杂页面或AI响应慢时会有加载提示。
2.  插件内存占用：力求高效，避免不必要的资源消耗。
3.  确保转换后的网页加载时间增加在可接受范围内。

## 7. 兼容性需求

1.  支持最新稳定版的 Chrome 浏览器。
2.  目标是兼容绝大多数主流网站，但动态内容复杂或结构特殊的网站可能存在挑战。

## 8. 安全性需求

1.  AI 辅助的样式生成主要依赖后端服务处理，用户输入的描述和页面结构会发送到后端。CSS 代码由后端生成并返回。
2.  承诺不主动收集与核心功能无关的用户个人身份信息或详细浏览历史数据。所有与样式应用相关的数据（如 hostname 和应用样式代码）存储在用户的 `chrome.storage.local` 或项目后端数据库（用于AI生成记录和样式管理）。
3.  定期审查和更新 AI 提示词策略，引导模型生成安全、有效的 CSS。

## 9. 后续计划

1.  用户反馈系统：收集用户对生成样式的满意度（评分系统已初步实现）。
2.  A/B测试不同的AI模型、提示词策略和样式生成方案。
3.  考虑开发其他浏览器的版本(如Firefox, Safari)。
4.  完善 "挂件模式" 的核心功能。
5.  实现用户账户系统与样式分享社区。
6.  增强自定义样式的编辑和管理能力。
