# StyleSwift 项目优化待办事项

## 一、代码结构与可维护性

- [ ] **模块化 `content.js`**: 将 `content.js` 拆分为更小的、功能单一的模块 (例如 `elementSelector.js`, `styleApplier.js`, `pageObserver.js`, `ratingUI.js`)。
- [ ] **模块化 `app.py`**: 使用 Flask Blueprints 将 `app.py` 中的不同功能模块（如用户认证、样式处理、等候名单管理）组织到不同的 Python 文件中。
- [ ] **常量管理**: 将 API 端点 URL、`chrome.storage` 的键名、消息 `action` 名称等字符串定义为常量，避免硬编码。
- [ ] **CSS 选择器生成健壮性 (`content.js`)**: 研究更稳定、更语义化的选择器生成方案，或允许用户在选择器失效时手动修正。

## 二、性能优化

- [ ] **优化 `content.js` DOM 操作**:
    - [ ] 优化 `getPageStructure` 函数，例如：只分析视口内的主要元素、限制遍历深度、避免对隐藏元素过多计算。
    - [ ] 对 `MutationObserver` 触发的 `checkAndApplyStyle` 进行防抖 (debounce) 或节流 (throttle) 处理。
- [ ] **优化后端 AI 调用**:
    - [ ] 确保 AI API 的 `timeout` 设置合理。
    - [ ] 在前端展示更明确的 AI 处理进度指示，或允许用户取消长时间等待的请求。
    - [ ] 研究是否有更快的 AI 模型或 API 接入点。

## 三、AI 提示词工程 (`app.py`)

- [ ] **持续优化提示词**: 根据实际生成的 CSS 质量，不断迭代和优化 `generate_prompt` 和 `generate_element_style_prompt` 中的提示词。
- [ ] **增强提示词上下文信息**: 在为单个元素生成样式时，提供其父元素、兄弟元素关键特征，或页面整体风格信息。
- [ ] **探索少量样本学习 (Few-shot Learning)**: 如果 AI API 支持，在提示词中加入高质量 CSS 生成范例。

## 四、用户体验 (UX)

- [ ] **改进错误处理与反馈**:
    - [ ] 当前后端发生错误时，给用户更友好和明确的提示信息。
    - [ ] AI 未能成功生成样式时，提供引导性建议。
- [ ] **完善加载状态指示**: 确保所有耗时操作（网络请求、AI 处理）都有清晰的加载状态反馈。
- [ ] **"全站点模式" 功能完善**: 进一步设计和实现全局挂件或自定义代码的管理与应用逻辑。
- [ ] **动态样式预览**: 考虑动态加载数据库中样式的预览图，或基于生成的 CSS 快速生成抽象预览。

## 五、安全性

- [ ] **加强输入清理**: 对所有用户输入（自定义描述、自定义CSS）和 AI 返回内容（CSS代码）在存储或应用前进行严格清理和验证。
- [ ] **严格内容安全策略 (CSP)**: 严格配置 `manifest.json` 中的 CSP，遵循最小权限原则。

## 六、数据库交互 (`app.py`)

- [ ] **查询优化**: 为频繁查询的字段（如 `styles.style_id`, `waitlist_entries.email`）确保已建立数据库索引。
- [ ] **事务管理**: 确保数据库操作（尤其是涉及多次读写的操作）在事务中正确执行。

## 七、扩展架构的健壮性

- [ ] **增强消息传递鲁棒性**: 为所有跨组件消息传递完善错误处理和重试机制。
- [ ] **关注状态同步**: 处理多个组件可能修改共享状态（如 `chrome.storage.local`）时的同步和潜在竞争条件。

## 八、"细节模式"样式合并逻辑

- [ ] **关注 CSS 特异性 (Specificity)**: 确保 AI 生成的元素样式能覆盖目标元素，且不意外影响其他元素。
- [ ] **优化样式合并函数**: 仔细审查 `remove_element_style` 和 `merge_styles` 的逻辑，避免破坏性修改。

## 九、其他潜在优化点

- [ ] **代码注释**: 补充和完善现有代码中的注释，特别是复杂逻辑部分。
- [ ] **单元测试/集成测试**: 为核心功能模块（前端和后端）编写测试用例。
- [ ] **依赖库版本管理**: 定期检查并更新项目依赖的库版本。
- [ ] **用户认证增强**: 如果需要更强的用户系统，考虑引入更成熟的认证方案 (如 OAuth, JWT)。
- [ ] **日志记录**: 优化前后端的日志记录，便于问题排查和系统监控。
