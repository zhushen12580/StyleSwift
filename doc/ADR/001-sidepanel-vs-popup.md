# ADR-001: Side Panel vs Popup

## 状态

已采纳

## 背景

Chrome 扩展有两种主要的 UI 形式：

1. **Popup**：点击扩展图标弹出的小窗口，宽度约 300px
2. **Side Panel**：浏览器侧边栏，宽度约 360-420px

## 决策

采用 **Side Panel**。

## 原因分析

### Popup 的问题

| 问题 | 影响 |
|------|------|
| 宽度受限（~300px） | 会话列表展示困难，工具调用详情无法完整展示 |
| 点击外部关闭 | 用户误点其他区域会丢失当前状态 |
| 每次点击重新加载 | 需要重新初始化，体验不连续 |
| 不支持持久化 | 无法保持 Agent 运行状态 |

### Side Panel 的优势

| 优势 | 说明 |
|------|------|
| 宽度充足 | 360-420px，会话列表、工具卡片、对话气泡都能舒适展示 |
| 持久化 | 切换 Tab 时保持打开，Agent 运行不受影响 |
| 用户体验 | 更像独立应用，而非临时弹窗 |

### 权限对比

两者权限需求相同（`sidePanel` 权限 vs 无额外权限），不影响用户感知。

## 后果

### 优点

- 更好的 UI 布局空间
- 连续的用户体验
- Agent 可以长时间运行

### 缺点

- 用户需要习惯点击图标后查看侧边栏，而非弹窗
- Side Panel 会占用浏览器宽度

### 技术影响

- Manifest 需声明 `sidePanel` 权限
- Service Worker 调用 `chrome.sidePanel.setPanelBehavior({ openPanelOnActionClick: true })` 实现点击图标打开

## 替代方案

**Popup + 新标签页**：简单交互用 Popup，复杂场景跳转新标签页。

放弃原因：切换成本高，体验不连贯，Agent 运行状态难以维持。
