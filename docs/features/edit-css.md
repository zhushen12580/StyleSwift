# 样式编辑

## 功能概述

精准修改已应用的 CSS，而非重新生成整段。

---

## 为什么需要

**场景**：用户说"标题再亮一点"

**方案对比**：

| 方案 | 做法 | 问题 |
|------|------|------|
| 重新生成 | 调用 `apply_styles` 注入新 CSS | 可能覆盖其他样式，token 消耗大 |
| 精准编辑 | 只修改颜色值 | 省时省 token，不破坏已有样式 |

---

## 设计目标

| 目标 | 说明 |
|------|------|
| 精准替换 | 只修改指定部分，不影响其他样式 |
| 省 token | 只传递旧片段和新片段，不重传整段 CSS |
| 安全性 | 必须精确匹配，避免误改 |

---

## 功能设计

### 输入

| 参数 | 说明 |
|------|------|
| old_css | 要替换的 CSS 片段（必须从当前样式中精确复制） |
| new_css | 新的 CSS 片段 |

### 工作流程

```
1. 验证 old_css 存在于当前样式中
2. 替换为新内容
3. 清空页面 CSS 栈
4. 重新注入合并后的 CSS
5. 更新存储
```

### 使用示例

```
当前样式：
.header { background: #1a1a2e; color: #e0e0e0; }
.content { color: #333; }

用户说："标题颜色改成白色"

Agent 调用：
old_css: ".header { background: #1a1a2e; color: #e0e0e0; }"
new_css: ".header { background: #1a1a2e; color: #ffffff; }"

结果：
.header { background: #1a1a2e; color: #ffffff; }
.content { color: #333; }
```

### 安全机制

**精确匹配要求**：
- `old_css` 必须与当前样式中的文本**完全一致**
- 包括空格、换行、分号

**为什么必须精确**：
```
当前样式：
.header { color: #e0e0e0; }

错误尝试：
old_css: ".header{color:#e0e0e0;}"  // 缺少空格
结果：匹配失败，返回错误

正确做法：
old_css: ".header { color: #e0e0e0; }"  // 精确复制
结果：匹配成功
```

### LLM 获取当前样式

通过工具 `get_current_styles` 获取当前会话已应用的完整 CSS，LLM 必须从该返回值中**精确复制**要修改的片段作为 `old_css`。

---

## 目标效果

| 场景 | 效果 |
|------|------|
| 微调颜色 | 只改颜色值，其他样式不变 |
| 调整字体 | 只改字号，布局不变 |
| 多次微调 | 每次只传少量文本 |

---

## 与 apply_styles 的关系

| | apply_styles | edit_css |
|---|---|---|
| 用途 | 添加全新规则 | 修改已有规则 |
| 输入 | 完整 CSS | 旧片段 + 新片段 |
| Token 消耗 | 较大 | 较小 |
| 使用时机 | 第一次添加样式 | 后续微调 |

**LLM 决策逻辑**（在 System Prompt 中引导）：
- 添加全新规则 → 用 `apply_styles`
- 修改已有样式 → 用 `edit_css`
- 全部重来 → 用 `apply_styles(mode=rollback_all)` 后重新生成

---

## 边界情况

| 情况 | 处理 |
|------|------|
| old_css 不匹配 | 返回错误，提示精确复制 |
| 当前无样式 | 返回错误，提示先用 apply_styles |
| 语法错误 | 返回错误，不应用 |
