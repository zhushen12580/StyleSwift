# 用户画像

## 功能概述

存储用户的偏好，以自由文本形式，供 LLM 生成样式时参考。

---

## 为什么需要

用户可能有长期偏好：
- "我喜欢高对比度"
- "讨厌圆角"
- "偏好深色背景"

如果每次都重新描述，体验很差。画像让 Agent"记住"用户偏好。

---

## 设计目标

| 目标 | 说明 |
|------|------|
| 灵活表达 | 不限于预设维度，用户可以说任何偏好 |
| 简单维护 | 用户可以更新、补充画像 |
| 按需使用 | 注入 System Prompt，LLM 自己理解 |

---

## 功能设计

### 存储形式

**自由文本**，非结构化 JSON：

```
用户画像示例：

"用户偏好高对比度的设计，深色背景配合浅色文字。不喜欢过于花哨的效果，
偏好简洁现代的风格。对字体敏感，希望正文字号不要太小。"
```

### 为什么用文本而非结构化

| | 结构化 JSON | 自由文本 |
|---|---|---|
| 表达能力 | 限于预设字段 | 任意偏好都能表达 |
| 扩展性 | 加字段需改代码 | 无需改动 |
| LLM 理解 | 需要解析字段 | 自然理解语义 |
| 示例 | `{"contrast": "high", "darkMode": true}` | "偏好高对比度深色模式" |

**结论**：结构化限制了表达，LLM 能理解自然语言，无需人为设限。

### 功能接口

| 操作 | 说明 |
|------|------|
| get_user_profile | 获取当前画像 |
| update_user_profile | 追加或更新画像内容 |

### 更新策略

**覆盖写入**（由 Agent 自行在“读取旧画像”的基础上整合后再写回）：

```
原有画像："偏好深色模式"

用户说："我不喜欢红色"
Agent 先 get_user_profile → 再调用 update_user_profile("偏好深色模式。不喜欢红色。")

更新后画像："偏好深色模式。不喜欢红色。"
```

实现说明：`update_user_profile` 的入参 `content` 是“完整画像内容”，会直接写入 `chrome.storage.local.userProfile`。

### 使用方式

画像注入到 System Prompt：

```
你是网页样式个性化智能体...

## 用户画像
{user_profile}

## 当前页面
域名：github.com
...
```

LLM 生成样式时自动考虑这些偏好。

---

## 目标效果

| 场景 | 效果 |
|------|------|
| 首次使用深色模式 | Agent 应用深色样式 |
| 用户说"标题太暗" | Agent 调整，并记录"深色模式下标题需高对比度" |
| 下次使用深色模式 | Agent 自动应用高对比度标题 |

---

## 设计考量

### 隐私

画像存储在本地（chrome.storage.local），不上传云端。

### 容量

画像应该简洁，建议控制在 1KB 以内。过长会影响 token 消耗。

### 清理

用户可以在设置中查看和清除画像。

---

## 边界情况

| 情况 | 处理 |
|------|------|
| 无画像 | 返回 `(新用户，暂无风格偏好记录)`；L1 注入时会省略 |
| 画像过长 | LLM 可能忽略部分内容（token 限制） |
| 冲突偏好 | LLM 根据上下文判断优先级 |
