# UI 界面设计

## 设计约束

Chrome Side Panel 宽度约 **360-420px**，高度跟随浏览器窗口。

所有界面采用**单列纵向流**布局，不做横向多列分割。

---

## 视图结构

| 视图 | 触发条件 | 说明 |
|------|---------|------|
| 安装欢迎页（独立页面） | 扩展首次安装 | 三步引导：功能介绍 → 置顶图标 → 风格选择（`extension/welcome/welcome.html`） |

### 操作确认浮层

样式应用后浮现：

```
┌─────────────────────────────────────────┐
│     [ ✓ 确认效果 ]    [ ↶ 撤销 ]        │  ← 单次 apply_styles(save)
│                 或继续输入              │
└─────────────────────────────────────────┘
```

多次样式应用（一次回复中发生多次 `apply_styles(mode="save")`）时，浮层变为“全部确认 + 撤销下拉”：

```
┌─────────────────────────────────────────┐
│ [ ✓ 全部确认 ]  [ ↶ 撤销最后一步 ▾ ]     │
│                 或继续输入              │
│  ────────────────────────────────────   │
│  （60s 超时进度条）                      │
└─────────────────────────────────────────┘
```

| 消失条件 | 说明（与实现一致） |
|---------|------|
| 点击确认 | 浮层消失，样式保留 |
| 点击撤销（单次） | 触发 `apply_styles(mode="rollback_last")`，浮层消失 |
| 撤销下拉（多次） | 可选：撤销最后一步 / 全部撤销 |
| 发新消息 | 隐式确认，浮层消失 |
| 60秒超时 | 隐式确认，浮层消失 |

### 输入区

| 状态 | 展示 |
|------|------|
| 空闲 | Picker 按钮 + 图片上传按钮 + 文本框 + 发送按钮 |
| 处理中 | 停止按钮（输入禁用，Picker 禁用，图片上传禁用） |
| 受限页面 | "此页面不支持样式修改"（整体置灰，Picker 禁用） |

**选中元素条**（条件出现）：当用户通过 Picker 选中页面元素后，在输入区上方显示一条预览条，包含已选元素的选择器/路径、清除按钮（×）；清除或发送带该上下文的请求后隐藏。

**图片附件条**（条件出现）：当用户选择上传图片后，在输入区上方显示图片预览列表与“一键清空”按钮；最多支持 **5 张**，大图会在本地压缩后再随消息发给 Agent（用于参考 UI/配色/风格等）。

### 元素选择器 (Picker)

用于在页面上点选一个 DOM 元素，将元素信息作为上下文随用户消息发给 Agent，便于「改这个元素的样式」等定向修改。

| 位置 | 说明 |
|------|------|
| 侧栏 | 输入框左侧的 Picker 按钮（十字准星图标），`title="选取页面元素"`。激活时按钮呈 active 状态。 |
| 选中条 | 选中后显示「选中元素条」，展示 `fullPath` / `selector` / `tag`，可点击 × 清除。 |

**页面内行为**（由 Content Script 实现）：

- 点击 Picker 后，当前标签页注入全屏遮罩（`z-index` 最高、`cursor: crosshair`），页面上出现跟随鼠标的高亮框（蓝色描边 + 半透明填充）。
- **选中**：点击高亮元素 → 提取元素信息，通过 `element_picked` 消息回传侧栏，关闭遮罩，侧栏显示选中元素条。
- **取消**：按 **ESC** 或**右键** → 发送 `picker_cancelled`，关闭遮罩，按钮恢复未激活。

**约束**：Agent 处理中、受限页面时不可开启 Picker。

**与 Agent 的联动**：选中元素后，发送消息时会附带该元素的 `selector`、样式摘要、子树文本等，供工具（如 `apply_styles`）精确定位目标。

## 安装欢迎页（独立页面）

扩展首次安装时自动打开的欢迎页（非 Side Panel 内视图），用于帮助用户快速上手：

- **第 1 步：功能介绍**：一句话改网页样式、保存风格技能等
- **第 2 步：置顶图标**：引导用户将扩展固定到浏览器工具栏（实现上以“延时自动完成/标记完成”简化流程）
- **第 3 步：设计风格**：展示预设风格卡片，可查看提示词并选择“使用此风格”

## 状态联动

| 状态 | 顶栏灯 | 对话区 | 输入区 | 技能区 |
|------|--------|--------|--------|--------|
| 就绪 | 🟢 | 正常 | 可输入 | 正常 |
| 处理中 | 🟡 脉动 | 流式输出 | 禁用 | 禁用 |
| 有样式生效 | 🟢 | 正常 | 浮层出现 | 正常 |
| API Key 无效 | 🔴 | 错误横幅 | 可输入 | 正常 |
| 网络错误 / API 错误 | 🔴 | 错误横幅（含重试） | 可输入 | 正常 |
| 受限页面 | ⚪ | 居中提示（说明内部页不可注入） | 置灰 | 置灰 |

## 设计原则

| 原则 | 说明 |
|------|------|
| 单列布局 | 适应 Side Panel 窄宽 |
| 信息层级 | 状态 → 会话 → 对话 → 输入 |
| 渐进展示 | 工具调用默认折叠，按需展开 |
| 即时反馈 | 状态灯、浮层、错误横幅 |
| 减少打断 | 确认浮层可超时消失，不强制操作 |
