# 网页样式美化器 Chrome 插件需求文档

## 1. 项目背景

随着互联网的发展,用户对网页美观度的要求越来越高。然而,许多网站的设计仍然简陋,无法满足现代用户的审美需求。同时，用户也希望能在网页上添加一些个性化的小工具或信息模块。为解决这些问题,我们提出开发一个Chrome插件,允许用户自定义网页样式和添加全局挂件。

## 2. 项目目标

开发一个Chrome插件,能够让用户：
1.  快速、简便地改变任何网页的视觉样式,以满足个人审美需求。
2.  在所有网页上添加自定义的功能或信息挂件。

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 全局挂件 (Global Widgets)
*   **目标:** 在所有访问的网页上添加用户自定义的功能或信息模块。
*   **作用范围:** 全局，默认应用于用户访问的所有网站。
*   **挂件管理:**
    *   用户可以添加、启用、禁用、删除挂件。
    *   可以调整挂件在页面上的大致位置（例如：左上角、右下角等）。
*   **创建/添加方式:**
    1.  **预设挂件:** 提供一系列内置挂件（如时钟、天气、待办事项、快速笔记等），用户可直接选用。
    2.  **AI生成挂件:** 用户通过自然语言描述挂件的功能和外观（例如："给我一个显示今天日期的挂件，背景透明，白色文字"），AI生成相应的挂件代码 (HTML/CSS/JS)。
    3.  **自定义代码挂件:** 用户可以直接输入 HTML, CSS, 和 JavaScript 代码来创建功能完全自定义的挂件。

#### 3.1.2 网页样式 (Web Styles)
*   **目标:** 美化特定网站的视觉外观。
*   **作用范围:** 特定域名，用户可为不同网站（域名）设置不同样式。
*   **样式管理:**
    *   用户可以为当前网站应用、修改、移除样式。
    *   可以管理已保存的网站样式。
*   **创建/应用方式:**
    1.  **预设样式模板:**
        *   **用户操作:** 用户从预设列表（如现代风、复古风、护眼模式、可爱风等）中选择一个风格模板。
        *   **后台逻辑:**
            *   接收用户选择的模板类型和当前页面的域名。
            *   查询数据库 `styles` 表，查找是否存在 `style_url` 匹配当前域名且 `style_type` 匹配所选模板的记录。
            *   **情况一 (找到记录):** 直接使用数据库中已有的 `style_code`。
            *   **情况二 (未找到记录):**
                *   触发核心 AI 样式生成流程（见下文），提示词中需包含所选风格模板对应的特定要求。
                *   将 AI 生成的 `style_code` 存储到数据库，关联当前域名和模板类型，并生成/分配唯一的 `style_id`。
        *   **应用与反馈:** 将获取到或新生成的 `style_code` 应用到用户当前页面。应用后，提供评分控件，允许用户对效果评分。评分数据更新到数据库对应的样式记录中（`total_ratings`, `total_score`, `average_rating`），以便后续排序或复用。
    2.  **AI生成样式 (自定义风格):**
        *   **用户操作:** 用户在输入框中通过自然语言描述想要的整体风格（例如："让这个新闻网站看起来更简洁，增加段落间距，使用柔和的蓝色调"）。
        *   **后台逻辑:**
            *   触发核心 AI 样式生成流程，使用用户的描述作为主要提示内容。
            *   （可选/后续优化：后台可尝试根据用户描述和生成的样式代码，为该样式归纳一个简单的风格标签）。
            *   将 AI 生成的 `style_code` 存储到数据库，关联当前域名，`style_type` 设为 'custom'，并生成/分配唯一的 `style_id`。
        *   **应用与反馈:** 将生成的 `style_code` 应用到用户当前页面。提供评分控件，用户评分后更新数据库记录。
    3.  **自定义 CSS:**
        *   **用户操作:** 用户在提供的编辑器中直接编写或粘贴 CSS 代码。
        *   **后台逻辑:**
            *   接收用户输入的 CSS 代码和当前域名。
            *   将用户输入的 `style_code` 存储到数据库，关联当前域名，`style_type` 设为 'custom-css'，并生成/分配唯一的 `style_id`。
        *   **应用:** 将用户提供的 CSS 代码直接应用到当前页面。
*   **核心 AI 样式生成流程 (用于预设模板和自定义风格):**
    1.  **提取结构:** 使用内容脚本 (`content.js`) 提取当前网页的 DOM 结构及关键 CSS 信息。
    2.  **简化处理:** 对提取的结构进行简化，移除不必要的节点和信息，保留骨架和关键样式，以适应 AI模型的输入限制并提高效率。
    3.  **构建提示词:** 结合简化后的页面结构、(如有)预设模板对应的提示词、以及(如有)用户的自然语言描述，构建最终的提示词发送给 AI 大模型。
    4.  **AI 生成:** 调用 AI 大模型（如 GPT-4, Claude等）生成 CSS 样式代码。
    5.  **代码处理:** 对 AI 返回的原始文本进行解析和清理，提取纯净的 CSS 代码。
*   **样式持续优化 (细节模式):**
    *   **激活时机:** 用户可以在应用了"网页样式"（通过上述任一方式）之后，或者**即使没有应用任何整体样式时**，都可以激活此模式。
    *   **通用流程:**
        1.  用户激活"细节模式"。
        2.  内容脚本启用元素选择功能，用户点击选择页面上的目标 HTML 元素。
        3.  弹出界面或侧边栏显示选中元素的基本信息，并提供输入框。
        4.  用户在输入框中用自然语言描述对该元素的具体样式修改要求（例如："把这个按钮的背景色改成绿色"，"让这段文字的字号再大一点"）。
    *   **后台处理逻辑 (根据是否已应用整体样式区分):**
        *   **场景A (未应用整体样式):**
            *   内容脚本提取选中元素的详细信息（包括其简化结构、计算样式等上下文）。
            *   后台构建提示词，包含元素详细信息和用户的修改要求。
            *   调用 AI 生成**仅针对该元素**的 CSS 属性代码 (e.g., `color: green; font-size: 16px;`)。
            *   后台将生成的属性代码与元素的选择器组合成完整的 CSS 规则。
            *   将这条新规则**添加**到与当前域名关联的样式记录中（如果不存在则创建，`style_type` 可设为 'details-only' 或 'custom'），或以其他方式独立存储。
            *   将更新后的（或仅包含此规则的）样式应用到页面。
        *   **场景B (已应用整体样式):**
            *   内容脚本提取选中元素的详细信息。
            *   后台获取当前域名正在应用的**完整** `style_code`。
            *   后台构建提示词，包含元素详细信息、用户的修改要求、以及**现有的相关样式作为上下文**。
            *   调用 AI 生成**优化后**的、针对该元素的 CSS 属性代码。
            *   后台使用**智能合并逻辑** (`merge_styles_intelligently`)，将新生成的属性代码更新到现有 `style_code` 中对应选择器的规则里（可能覆盖旧属性、添加新属性）。
            *   将修改后的**完整** `style_code` 更新回数据库中对应的样式记录。
            *   将更新后的完整样式应用到页面。
    *   **效果预览与保存:** 样式修改应能实时或准实时预览。用户确认后，修改被持久化保存到数据库。

### 3.2 插件交互

1.  **插件图标:** 点击Chrome工具栏中的插件图标，弹出插件操作界面。
2.  **弹出界面:**
    *   清晰地区分"全局挂件"和"网页样式"两大功能区域。
    *   提供对应功能的设置和操作入口。

### 3.3 扩展功能 (保留，待后续迭代细化)

1.  **样式/挂件分享与市场:** 用户可以分享自己创建的样式或挂件，也可以浏览和使用他人分享的内容。
2.  **网页类型识别:** 自动识别网页类型（新闻、社交、电商等），为"网页样式"推荐更合适的预设模板或AI生成策略。
3.  **性能优化:** 实现样式/挂件缓存，优化加载速度和资源占用。
4.  **跨设备同步:** （可选）通过用户账户同步用户的挂件和样式设置。

## 4. 技术需求

1.  **Chrome插件开发:** 使用HTML, CSS, JavaScript,flask开发插件界面和功能，遵循最新开发规范。
2.  **网页结构分析:** 开发能够准确提取网页DOM结构、识别关键元素的模块（支持"AI生成样式"和"细节模式"）。
3.  **AI集成:**
    *   选择合适的AI大模型（具备代码生成能力，如 GPT-4 或同等模型）。
    *   开发与AI模型的接口，实现：
        *   自然语言到挂件代码 (HTML/CSS/JS) 的转换。
        *   网页结构+自然语言描述 到 CSS 样式代码的转换。
        *   选中元素+自然语言描述 到 元素特定CSS 的转换。
4.  **样式/挂件注入与渲染:**
    *   开发能够动态、安全地注入和渲染 CSS 及 JS (用于挂件) 的模块。
    *   确保注入的样式和脚本优先级，同时避免破坏网页原有功能和布局。
    *   挂件需要考虑沙箱环境或安全措施，防止恶意脚本。
5.  **安全性:**
    *   对用户输入的自定义代码 (CSS/JS) 进行必要的安全检查或限制。
    *   AI生成的代码也需要进行安全评估。
    *   挂件运行环境应与原页面适当隔离（如 Shadow DOM）。

## 5. 用户界面需求

1.  **插件弹出窗口:**
    *   简洁直观，易于导航至"全局挂件"和"网页样式"功能。
    *   **挂件管理界面:** 列表显示已添加挂件，提供启用/禁用、删除、编辑（代码或描述）、位置调整选项。提供添加新挂件入口（预设、AI、自定义代码）。
    *   **样式管理界面:** 显示当前域名应用的样式信息。提供应用新样式入口（预设、AI、自定义CSS）。提供进入"细节模式"按钮。提供移除当前样式选项。
    *   **细节模式界面:** （可能需要结合内容脚本）提供元素选择提示，显示选中元素信息，提供自然语言输入框，显示预览效果，提供保存/取消按钮。
2.  **（可选）设置页面:**
    *   管理所有已保存的网站样式。
    *   管理所有创建的挂件。
    *   插件偏好设置（例如：默认挂件位置等）。

## 6. 性能需求

1.  样式转换/应用响应迅速（目标 < 3秒）。
2.  挂件加载不应显著拖慢页面加载速度。
3.  插件和注入脚本的内存占用合理（目标 < 50-100MB）。
4.  应用样式或挂件后，页面交互流畅性不应受明显影响。

## 7. 兼容性需求

1.  支持最新稳定版的 Chrome 浏览器。
2.  网页样式功能需兼容主流网站（目标 > 90%）。
3.  挂件功能需能在绝大多数网站上正常显示和运行。

## 8. 安全性需求

1.  所有操作在用户浏览器本地完成，不上传用户浏览内容。
2.  AI生成请求仅发送必要的脱敏信息（如页面结构、用户描述），不发送页面具体内容或个人数据。
3.  不收集用户个人身份信息或详细浏览历史。
4.  对用户输入的代码和AI生成的代码执行进行安全限制，防止跨站脚本（XSS）等攻击。
5.  定期更新和评估AI模型安全性。

## 9. 后续计划

1.  实现 MVP (Minimum Viable Product)，包含核心的挂件添加（预设/自定义代码）和样式应用（预设/自定义CSS）。
2.  逐步集成 AI 生成能力。
3.  开发"细节模式"功能。
4.  完善用户反馈渠道。
5.  根据用户反馈和数据分析，迭代优化功能和体验。
6.  探索扩展功能（分享、类型识别等）。
